# Plugins and Skills

Plugins extend Webcmd with adapter packs. Skills teach agents how to use those adapters, crawl new sites, and repair broken commands.

## Plugins

Use plugins when a set of commands should travel together:

- Internal company dashboards.
- Customer support tools.
- Research workflows.
- Community adapter packs.
- Experiments outside the core Webcmd registry.

Ask an agent to install one:

```text
Install the Webcmd plugin from github:my-org/webcmd-acme, list the commands it adds, and run one smoke check.
```

Ask an agent to create one:

```text
Create a Webcmd plugin called `webcmd-acme` from my private Acme adapters. Include a manifest, README examples, and keep the commands discoverable with `webcmd list -f json`.
```

Common plugin commands:

```bash
webcmd plugin list
webcmd plugin update --all
webcmd plugin uninstall <name>
```

Create a plugin scaffold:

```bash
webcmd plugin create <name>
```

You can run those yourself, but they are usually part of an agent task.

## Skills

Skills are instructions for agents. They help the agent choose the right workflow instead of guessing.

Webcmd has two install surfaces:

| Agent surface | Recommended install |
| --- | --- |
| Codex | Install the Webcmd plugin from the Codex plugin marketplace. |
| Claude Code | Install the Webcmd plugin from the Claude Code plugin marketplace. |
| Other skill-aware agents | Install the Webcmd discovery skill with `npx skills add agentrhq/webcmd --skill webcmd`. |

Install the CLI separately with `npm install -g @agentrhq/webcmd`. The skill or plugin teaches the agent how to use the `webcmd` binary; it should not be treated as the CLI package itself.

Common skills:

| Skill | Purpose |
| --- | --- |
| `webcmd-usage` | Orient an agent to commands, formats, strategies, plugins, and external CLIs. |
| `webcmd-browser` | Drive a real browser ad hoc. |
| `webcmd-adapter-author` | Build or extend a reusable adapter. |
| `webcmd-autofix` | Repair adapters after site changes. |
| `webcmd-browser-sitemap` | Use recorded site navigation knowledge while browsing. |
| `webcmd-sitemap-author` | Capture reusable sitemap knowledge. |
| `smart-search` | Route research/search requests to the right adapter. |

Prompt example:

```text
Use the Webcmd adapter-author skill to create this command. If the site changes while testing, use the autofix workflow before finishing.
```

## Installing Skills for Agents

If your agent supports skills, install the Webcmd discovery skill. Then tell the agent to load the runtime workflow it needs.

```text
Before working on this site, load the Webcmd usage workflow and prefer an existing adapter if one exists.
```

## Recommended Agent Behavior

Agents should usually follow this order:

1. Discover commands with `webcmd list -f json`.
2. Use an existing adapter when one fits.
3. Explore the browser only when no adapter exists or when debugging.
4. Author or repair an adapter when the workflow should be repeated.
5. Package related commands as a plugin when they should be shared.

This keeps browser work exploratory and turns repeated work into stable commands.
